AI Operational System in Production (2023)

Eric Fricke · Production system built inside a small agency · Mid 2023
Contact: fricke.ea@gmail.com

This is a production AI workflow system built inside a small digital agency after losing a sales director. It began as a CRM augmentation project and expanded into an event-driven operational system for onboarding, content generation, and sales augmentation.

1. Origin

The initial idea was a CRM “wiki layer” for junior sales staff: lead history, objections, prior interactions, and context.

The project brief was drafted in ~90 minutes during an operational discussion with ownership.

During that process, scope expanded into:

  • lead scoring
  • sentiment analysis
  • workflow automation
  • AI content generation
  • onboarding automation
  • feedback optimization loops

The system shifted from CRM extension → workflow orchestration layer.

2. System Overview

Built as an ExpressionEngine module acting as an orchestration layer over CRM and external AI APIs.

  • event-driven workflows
  • API-based AI generation
  • cron-based orchestration
  • human review gates
  • modular system design for provider swapping

3. Event-Driven Onboarding

A payment event triggered full provisioning:

  • CRM record creation
  • Slack channel provisioning
  • workflow initialization
  • onboarding form generation

After onboarding submission:

  • website scraping (Python)
  • content structuring into training objects
  • AI model adaptation via OpenAI API
  • initial content generation pipeline

Target behavior: client could sign up at night, system ready for review next morning.

4. Content System

Each client had a structured content generation pipeline:

  • blogs
  • social posts
  • SEO content
  • website copy

Context sources:

  • scraped historical websites
  • previous generated content
  • brand preferences
  • stored client data

~23 prompts were used across the marketing pipeline.

5. Supervisory Loop

A higher-level model aggregated cross-client performance data.

  • engagement metrics
  • campaign performance
  • human edits
  • accept/reject signals

Weekly cycle:

  • generate recommendations per client
  • client system applies or rejects
  • human edits feed back into future behavior

This created a recursive feedback system between generation and human correction.

6. Sales System

  • lead enrichment
  • lead scoring
  • sentiment overlays
  • automated follow-ups
  • CRM augmentation

7. Constraints

  • API cost scaling
  • workflow synchronization complexity
  • prompt orchestration overhead
  • provider dependency
  • system complexity growth over time

8. Outcome

  • ~60 day onboarding → ~3 weeks
  • <2 hour support response across ~90 clients
  • ~4x revenue growth in ~70 days
  • reduced operational load without headcount scaling

9. Key Insight

AI was not used as a tool inside an existing workflow. It was embedded into the workflow architecture itself.

This changed:

  • delivery speed
  • organizational structure
  • coordination overhead
  • pricing model
  • operational throughput
